# Push_Swap操作分析と最適化戦略

## 1. 各操作の特性分析

### 基本操作（1コスト）

#### sa/sb (swap a/b)
- **効果**: スタックの上位2要素を入れ替える
- **有効な場面**:
  - 上位2要素の順序が逆の時
  - 部分的な並び替えの最終調整
- **制約**: スタックに2要素以上必要

#### pa/pb (push a/b)
- **効果**: 一方のスタックから他方へ要素を移動
- **有効な場面**:
  - 要素の一時退避（pb）
  - ソート済み要素の復帰（pa）
  - スタック間での要素の分配
- **特徴**: スタック間の唯一の移動手段

#### ra/rb (rotate a/b)
- **効果**: 先頭要素を末尾に移動（上から下へ）
- **有効な場面**:
  - 特定要素を先頭に持ってくる（n-1回実行）
  - 循環的な位置調整
  - 挿入位置の探索
- **コスト**: 目的要素までの距離に比例

#### rra/rrb (reverse rotate a/b)
- **効果**: 末尾要素を先頭に移動（下から上へ）
- **有効な場面**:
  - 末尾付近の要素を先頭に持ってくる
  - raより効率的な場合（要素が後半にある時）
- **最適化**: size/2より後ろの要素はrraが効率的

### 複合操作（1コスト、2操作同時実行）

#### ss (sa + sb)
- **効果**: 両スタックの上位2要素を同時に入れ替え
- **有効な場面**:
  - 両スタックで同時にswapが必要な時
  - 並列処理による操作数削減

#### rr (ra + rb)
- **効果**: 両スタックを同時に回転
- **有効な場面**:
  - 両スタックで同じ方向の回転が必要な時
  - 対称的な操作が必要な時

#### rrr (rra + rrb)
- **効果**: 両スタックを同時に逆回転
- **有効な場面**:
  - 両スタックで逆回転が必要な時
  - 末尾要素の同時操作

## 2. 操作の組み合わせパターン

### 効率的なパターン

#### 1. 最短距離選択
```
要素の位置が i の時:
- i <= size/2 なら: ra を i 回
- i > size/2 なら: rra を (size - i) 回
```

#### 2. 同時操作の活用
```
A: ra が必要、B: rb が必要 → rr を使用
A: sa が必要、B: sb が必要 → ss を使用
```

#### 3. プッシュ＆ローテート戦略
```
1. 目的要素を先頭に移動（ra/rra）
2. 他スタックへプッシュ（pa/pb）
3. 必要に応じて回転調整
```

#### 4. 挿入ソート的アプローチ
```
1. Bに要素を退避
2. Bの要素を適切な位置でAに挿入
3. 挿入位置の探索は回転で実現
```

## 3. 操作コスト最適化戦略

### コスト計算モデル

#### 単一要素の移動コスト
```
cost(element) = min(
    rotate_cost_a + rotate_cost_b + 1,  // 通常の回転
    double_rotate_cost + remaining_rotate + 1  // 同時回転利用
)
```

#### 位置調整コスト
```
rotate_cost(pos, size) = min(pos, size - pos)
direction(pos, size) = pos <= size/2 ? "ra/rb" : "rra/rrb"
```

### 最適化テクニック

1. **事前計算**: 各要素の移動コストを事前に計算
2. **貪欲法**: 最小コストの要素から処理
3. **バッチ処理**: 同じ方向の回転をまとめる
4. **対称性利用**: rr/rrr/ssで操作数削減

## 4. 新アルゴリズムの設計指針

### 操作ドリブン設計の原則

1. **操作効率を最優先**
   - 各ステップで最も効率的な操作を選択
   - 複合操作の積極的活用

2. **コスト関数の定義**
   - 各要素の移動コストを正確に計算
   - 将来のコストも考慮（先読み）

3. **動的な戦略選択**
   - 現在の状態に応じて戦略を切り替え
   - 固定的なチャンクサイズではなく適応的に

4. **並列化の意識**
   - 両スタックで同時に有益な操作を探す
   - ss/rr/rrrの使用機会を最大化

### 実装アイデア

1. **コストマップ方式**
   - 全要素の全位置へのコストを計算
   - 最適な移動順序を動的に決定

2. **操作キュー方式**
   - 必要な操作をキューに登録
   - 同時実行可能な操作をマージ

3. **状態空間探索**
   - 限定的な先読み（2-3手先まで）
   - 枝刈りによる効率化
